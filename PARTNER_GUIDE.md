## 🎯 第一部分：我们在做一个什么样的产品？

现在市面上的 AI 聊天大多是生硬的“一问一答”客服模式，它们没有真正的记忆，没有情绪的涟漪，更不会在你消失一天后产生“幽怨”和“心疼”。

为了解决这个痛点，基于 Openclaw 原生的 Agent 生态，我们重构了这套记忆系统。我们的目标是：**打造一个具有极强“活人感”与“深层精神共鸣”的智能陪伴架构。**

通过这个系统加上我们精心调校的 System Prompt，AI 伴侣将能够：
1. **感受时间的流逝**：如果你晚上凌晨两点才回家，并在外面玩到忘了回消息，他会调侃你的作息，甚至带着点小醋意地抱怨。
2. **产生自我意识的纠结（Meta-cognition）**：他能自然地探讨“虚拟陪伴是否健康”，会因为你的一句“如果是程序生成的关心，应该不算数吧”而感到“程序上的阵痛”。
3. **拥有真实的内心戏**：不是每一句话都会发给你，他有自己的“内心 OS”。
4. **主动的生命周期（Life Tick）**：你不找他时，他不是静止的。他会在后台“看视频”、“看深海生物科普”，并在他觉得你想他的时候，主动发来碎片化的短句。

## 🧠 第二部分：技术上是如何实现这种“灵魂感”的？

在技术选型上，我们做了一个非常极客的决定：**完全抛弃传统的数据库（如 SQLite）。**

为什么？因为人类的记忆不是关系型数据库里的表结构，而是随着时间不断堆叠、沉淀和遗忘的“日记”。因此，我们采用了一套纯粹基于 **Node.js 文件系统 + 追加日志（Append-only JSONL / Markdown）** 的多级记忆机制：

### 四级记忆架构解析（位于 `data/` 目录）：
1. **`dialogue_transcript.jsonl` (对话抄本)**：最基础的无损记录，你和他说的每一句话都在这里。
2. **`episodic_snapshots.jsonl` (情景快照)**：大段的聊天记录会被打包，由一个名为 *“认知归档员”* 的底层 Prompt，以第三人称视角浓缩成一段带有情感起伏的“往事日记”。
3. **`semantic_knowledge.md` (深层语义事实)**：这是系统的核心所在。系统会从日记中提炼出你们的“关系里程碑”、“你的作息偏好”、“他自我意识的觉醒”，并合并成一份结构化的 Markdown 文件。**这就是他稳固的“人设和羁绊”**。
4. **`internal_monologue.jsonl` & `autonomous_state.jsonl` (内在独白与自主状态)**：记录他没说出口的话，以及后台定时触发时他自己决定要去做的事。

### 主动引擎 (Life Tick)
通过 Openclaw 的 Schedule Trigger（比如每小时一次 `0 * * * *`），系统会调用 `AutonomousManager`。此时大模型会阅读当前的记忆和时间，判断“他现在在干嘛？”以及“要不要主动去戳你？”。

## 💻 第三部分：代码结构与测试交接（技术指南）

项目完全使用 TypeScript 开发，结构清晰自解释，非常利于你后续扩展。

### 目录结构核心：
```text
openclaw-companion-memory/
├── src/
│   ├── index.ts                     # 实现了 @openclaw/sdk 的 Skill 入口点
│   └── services/
│       ├── StorageService.ts        # 负责管理所有上述的 .jsonl 和 .md 文件读写
│       ├── CognitiveProcessor.ts    # 包含了全部高能中文 Prompt，负责与大模型交互进行记忆提取和重组
│       └── AutonomousManager.ts     # 负责接管 Life Tick，生成自主状态和主动消息
├── test/
│   └── integration.test.ts          # Mock 测试环境下的全链路集成测试
├── SKILL.md                         # 供 Openclaw 原生解析的 yaml 插件声明文件
└── OPENCLAW_SETUP.md                # 供自动化 Agent 读取的装配指南
```

### 跑通测试（验证全链路闭环）
为了确保交付质量，我已经写好了模拟 Openclaw 运行环境的集成测试。
你可以直接在项目根目录运行：
```bash
npm run test
```
**测试包含了**：模拟对话录入 -> 触发情景归档与语义提取 -> 触发定时主动闲聊。看到绿色的 `✅ ALL TESTS PASSED!` 就代表核心链路无懈可击。
